Building a Component System with Tailwind

How to leverage Tailwind's utility classes to build reusable components.

Contents

Building a Component System

Tailwind CSS is a utility-first framework, but that doesn’t mean you can’t build reusable components. In fact, it encourages it!

The Button Component

Here is how you might define a simple button component in Astro:

---
// src/components/Button.astro
const { href } = Astro.props;
---
<a href={href} class="px-4 py-2 bg-pearl-500 text-ink-950 rounded hover:bg-pearl-400 transition-colors">
  <slot />
</a>

Using @apply

Sometimes you want to extract classes to keep your HTML clean. You can use @apply in your CSS:

.btn-primary {
  @apply px-4 py-2 bg-pearl-500 text-ink-950 rounded hover:bg-pearl-400 transition-colors;
}

However, the Tailwind team generally recommends keeping utilities in your HTML for better maintainability.

Note: Always prefer composition over abstraction when starting out.

Conclusion

Tailwind gives you the building blocks. It’s up to you to assemble them into a coherent system.
